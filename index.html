Welcome to the SQL Field Guide

This is a community-built reference guide for SQL learners who have some coursework or hands-on experience and want clearer mental models for the concepts that textbooks or online courses tend to move past too quickly.

Work in progress This guide started as a personal study tool and is actively being developed. Content is added topic by topic. If something is missing, incomplete or could be explained more clearly, contributions are welcome.

Who is this for

Beginner Students in the first SQL courses or those building foundational knowledge. No prior database experience assumed.

Intermediate Learners who can write SQL confidently but want to understand what the database engine is doing: transactions, locking, joins and query behavior. Suitable for coursework, college studies and early professional work.

Advanced Practitioners who use SQL professionally and want to go deeper into performance, Concurrency patterns and query optmization.

How to use this field guide

Use the sidebar to the left to find a topic. Each topic page has three sections:

Explainer - the mental mode and plain-English reasoning behind the concept.

Interactive - a hands-on widget to explore the concept yourself.

Reference - a compact cheat sheet to return to after you understand the topic.

Limitations

This guide focuses on Microsoft SQL Server and T-SQL unless otherwise noted. Examples may not transfer directly to MySQL, PostgreSQL or other dialects without adjustment. Where behavior differs significantly between databases, a note is included.
