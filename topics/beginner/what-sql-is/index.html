What SQL Is & How a Query Runs

Before this topic: nothing — this is the starting point for the Beginner tier. No prior SQL knowledge required.

A database is a collection of tables and a table is just a structured list — think of a well-organized spreadsheet. Each row is one record; each column is one property that record has.

A query is a sentence with a strict grammar

SELECT says which columns you want to see. FROM says which table to look in. Every query you write in this Beginner tier builds on just those two ideas, plus a few more clauses you'll meet on the next few pages.

SELECT

Lists exactly which columns you want back. Leave one out, and it simply won't appear in your results — nothing is selected by default.

FROM

Tells SQL which table to look in. Every query needs exactly one starting table (you'll combine more than one with JOIN later.)

SQL is written SELECT...FROM... , but the engine actually works it out the other way around — it figures out which table you're using (FROM) before it even looks at what you asked to see (SELECT). It won't matter much yet, but once you've learned WHERE and GROUP BY, this processing order explains a few things that otherwise feel backwards.

Every column has a type

SQL Server enforeces a data type on every column and it affects how you can query it later — comparing text to a number, or running a data function on a text column, are common early mistakes.

TypeHoldsWhy it matters
INTWhole numbersYou can do math on it
VARCHARTextNeeds quotes in a query; no math
DATECalendar datesCompares chronologically, not alphabetically
DECIMALPrecise numbers with fixed decimal placesUsed for money — avoids the rounding errors FLOAT can introduce

Basic query shape

SELECT column1, column2
FROM table_name;

Column order in SELECTdetermines column order in the result — it doesn't have to match the table's actual column order.

Select every column

SELECT * FROM table_name;

Useful for quick exploration. Avoid it in real application code — naming columns explicitly is safer if the table's structure changes later.
